Hugo: 基于 Hugo+GithubPage 搭建静态网站

Posted at — Mar 19, 2022

[TOC]

通过Hugo和GithubPage搭建简易的日志记录的静态网站。本文记录自己搭建的过程和踩过的坑。

Hugo The world’s fastest framework for building websites. Hugo is one of the most popular open-source static site generators. With its amazing speed and flexibility, Hugo makes building websites fun again.

Install Hugo

Hugo的下载地址,根据自己的操作系统选择不同的系统版本。本机选择hugo_extended_0.95.0_Windows-64bit.zip。

  1. 解压至本机放置软件源包的文件夹
  2. 将hugo.exe加入path环境中
  3. 启动cmd, hugo version查看hugo的版本

搭建Hugo项目

  • 初始化项目
# 初始化一个hugo项目 hugo new site [项目名称]
hugo new site awsomework_log
  • 选择适合你的hugo themes或者自己写

Hugo的基础语言是Golang，所以其themes的模板基于golang,同时Hugo也有属于自己自定义的模板语言。

由于只需要记录日常的技术Trace以及技术BUG,所以我选择较为简单的themes-ezhil

  1. 下载themes-ezhil
  2. 将zip包解压至awsomework_log中,替换其中相同的所有

  2. 最为重要的是config.toml配置文件的修改

{{- $.Scratch.Set "isGitalk" false -}}

<!-- Check if content has flag to enable/disable disqus explicitly -->
{{- if and (isset .Params "key") (eq .Site.Params.enableGitalk true) -}}

{{- end -}}
{{- if eq ($.Scratch.Get "isGitalk") true -}}

{{- end -}}

部署github page

  • 建立github page repo, 详细参考
  • 关联本地站点到github page repo
cd awsomework_log
mkdir public
cd public
git init .
git checkout -b main
git remote add origin git@github.com:xxx/xxx.io.git
git pull origin main 

# hugo 生成public命令
# 其中--theme的值一定要和config中的theme值一致
# --baseUrl 一定要要和自己的github page 的地址一致
# 其实就是将public中配置对应字段进行替换
hugo --theme=codemax --baseUrl="https://hzzhu92.github.io/"

# git add .
# git commit -m "comment"
# git push origin main
